## Context

现有 `_calculate_chapter_distribution` 算法（对所有题材使用相同比例）：

```
3部: 25% - 50% - 25%  (起源-深渊-决战)
4部: 20% - 30% - 30% - 20%  (双峰中段)
5部+: 20% - 20% - 20% - 20% - 20%  (完全均匀)
```

这导致：
- 玄幻 5 部 → 每部约 200 章，全部"发展/深渊"，升级爽感节奏被压平
- 历史 3 部 → 比例 OK，但 4 部以上全部均匀，政争/战争的大高潮无法集中在中段体现
- 都市 5 部 → 应该是波浪形，但被压成均匀，单调乏味

## Decisions

### 1. 题材章节比例模型

**玄幻小说 — 倒V形（境界递进）**

每突破一个大境界 = 一部，境界越高体量越大（副本更多、冲突更复杂）：

```
5部: 10% - 20% - 25% - 20% - 15%
4部: 12% - 24% - 28% - 24% - 12%
3部: 15% - 35% - 30% - 20%
```

**历史小说 — 幂函数形（政治周期）**

政治博弈越到后期越复杂，中间部承担最多叙事重量：

```
3部: 25% - 50% - 25%
4部: 20% - 30% - 30% - 20%
5部: 20% - 20% - 25% - 20% - 15%  (中后部最重)
6部+: 递减，首尾各10%，中间各~16%
```

**都市小说 — 波浪形（人生阶段）**

主角经历多个人生阶段，中间段多线并行体量最大：

```
3部: 20% - 40% - 40%
4部: 15% - 25% - 30% - 30%
5部: 15% - 20% - 25% - 20% - 20%
```

**混合/默认 — 黄金分割（经典叙事）**

```
3部: 25% - 50% - 25%
4部: 20% - 30% - 30% - 20%
5部+: 首尾各20%，中间平分60%
```

### 2. 题材叙事阶段标签

pacing_guide 中每部的标签必须符合该题材的叙事习惯：

**玄幻标签**（境界递进）：
- 第1部: 「起源卷」「觉醒卷」「筑基卷」
- 中间部: 「崛起卷」「风云卷」「乱世卷」
- 最后部: 「巅峰卷」「飞升卷」「终章卷」

**历史标签**（政治周期）：
- 第1部: 「崛起篇」「奠基篇」「初锋篇」
- 中间部: 「争锋篇」「博弈篇」「权倾篇」「烽火篇」
- 最后部: 「终焉篇」「大一统篇」「新朝篇」

**都市标签**（人生阶段）：
- 第1部: 「起步篇」「草莽篇」
- 中间部: 「崛起篇」「风云篇」「巅峰篇」
- 最后部: 「收官篇」「后传篇」

### 3. Pacing_guide 动态内容

每部的 pacing_guide 描述应包含该部的**叙事重点**，而非单调的"起源/深渊/决战"：

```python
# 玄幻示例（5部，第3部）：
"第3部（中期高潮）：分配 250 章。情节要求：主角境界稳步提升，多方势力角逐，
  中段需有1-2个大型副本（每个50-80章），小高潮每20-30章一次，
  结尾为全书最大爽点之一（灭敌盟主/天骄榜首）。"
```

### 5. 题材路由

```python
def _get_distribution_for_genre(self, genre: str) -> str:
    genre_lower = genre.lower()
    if "xuanhuan" in genre_lower or "玄幻" in genre_lower:
        return "xuanhuan"
    elif "history" in genre_lower or "历史" in genre_lower:
        return "historical"
    elif "dushi" in genre_lower or "都市" in genre_lower:
        return "dushi"
    else:
        return "default"
```

## Risks / Trade-offs

- [Risk] 题材路由使用 substring 匹配，可能误判（如"都市玄幻"）
- [Mitigation] 只在 `genre` 完全匹配时触发特定分布，否则回退 default

- [Risk] 混合题材小说（历史+玄幻）如何处理
- [Mitigation] 混合类型回退到 default，使用黄金分割；前端创建时限制只选一种主类型
